History

TCP is a window based protocol, i.e. it implements a window based flow control mechanism where the current window size restricts the amount of unacknowledged data that can be transferred between a TCP sender and receiver. Originally this maximum window size awnd(advertised windows) was advertised by the receiver.

1980s - new congestion control mechanisms introduced to prevent congestion in the internet caused by TCP senders overrunning resources of the network.

1988 - Tahoe TCP released, included 3 congestion control algorithms: slow start, congestion avoidance and fast retransmit. Two new variables cwnd(congestion window) and ssthresh(slow start threshold) introduced.

1990 - Reno TCO released, included the fast recovery algorithm

TCP sender's window size w = min (cwnd, awnd)
awnd prevents the sender from overrunning the resources of the receiver. cwnd prevents the sender from sending more data than that the network can handle. cwnd can be set according to the packet loss condition which can be detected either via a time-out mechanism or via duplicate ACKs.
Each packet has a timer (RTO Retransmission timeout) associated with it, and when that timer expires the packet is retransmitted. RTO should be similar to that of an RTT(Round trip time) which is usually calculated using algorithms such as Karn.
