command syntax: openstack quota <action/operation> <--flags> <values> <project-uuid>

As the name suggests, Quotas refer to the limit on the no. of resources that can be utilized by a project

cumulus@server01:~$ openstack project list
+----------------------------------+--------------+
| ID | Name |
+----------------------------------+--------------+
| 23595ea6e5ed4201914e3831c12820ae | firstProject |
| 857b18af8e4540ab94d2f41b220a91b3 | service |
| 9a4ce13914f0444789153f632722a42a | admin |
| b9cf72612b7041c5a72e483551d07e93 | demo |
+----------------------------------+--------------+

#to check the quotas associated with the project 'firstProject'

cumulus@server01:~$ openstack quota show 23595ea6e5ed4201914e3831c12820ae
+----------------------+----------------------------------+

| l7_policies | None |
| listeners | None |

| location | None |
| name | None |
| networks | 100 |
| pools | None |

cumulus@server01:~$

let's filter the quota for instances

cumulus@server01:~$ openstack quota show 23595ea6e5ed4201914e3831c12820ae | grep instances
| instances | 10 |
cumulus@server01:~$

#lets change the quota for maximum no. of 'instances' from 10 to 20

cumulus@server01:~$ openstack quota set --instances 20 23595ea6e5ed4201914e3831c12820ae

let's verify the change we made

cumulus@server01:~$ openstack quota show 23595ea6e5ed4201914e3831c12820ae | grep instances
| instances | 20 |
cumulus@server01:~$
